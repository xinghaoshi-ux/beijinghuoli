# 公共端筛选与排名需求说明 v2.1

## 1. 本轮目标

在现有比赛结果、球队排名、个人排名基础上，增加更适合长期多队比赛的数据查询能力：

```text
按年月查询
按比赛日期查询
按球队查询
按年龄组查询
排名页支持按球队筛选，但保留全局排名名次
```

本轮同时将赛事名称统一为：

```text
北京市活力网球交流系列赛
```

## 2. 比赛结果查询

比赛结果需要支持以下筛选：

| 筛选项 | 说明 |
|---|---|
| 搜索 | 搜索球员、球队、项目、场地或备注 |
| 月份 | 格式 `YYYY-MM` |
| 比赛日期 | 格式 `YYYY-MM-DD` |
| 球队 | 支持约 20 个队的可搜索下拉 |
| 年龄组 | 例如 `70岁组`、`90岁组`、`110岁组` |

默认展示：

```text
最近一个有比赛结果的比赛日期
```

## 3. 比赛日期下拉菜单

公共端必须提供比赛日期下拉菜单。

数据来源：

```text
GET /api/v1/public/filter-options
```

交互规则：

- 月份未选择时，日期下拉展示全部有比赛的日期。
- 月份已选择时，日期下拉只展示该月份内有比赛的日期。
- 日期选中后，比赛结果只展示该日期的比赛。
- 日期和月份都清空时，恢复默认最近比赛日。

## 4. 个人排名

个人排名默认展示最新全局总榜。

规则：

- 排名页不展示月份、比赛日期、年龄组筛选。
- 可通过顶部球队下拉筛选某支球队的球员。
- 筛选球队后只缩小展示范围，不重新计算名次。
- `rank` 始终保留全局总榜名次。

排序规则保持：

```text
1. 个人胜盘数
2. 个人胜局数
3. 个人净胜局
4. 出场次数
5. 仍相同则并列
```

排名显示规则：

```text
并列后不跳号，例如 1、1、2、3。
```

## 5. 球队排名

球队排名默认展示最新全局总榜。

规则：

- 排名页不展示月份、比赛日期、年龄组筛选。
- 可通过顶部球队下拉筛选某支球队。
- 筛选球队后只缩小展示范围，不重新计算名次。
- `rank` 始终保留全局总榜名次。

排序规则保持：

```text
1. 团体胜场
2. 胜盘数
3. 胜局数
4. 净胜局
5. 仍相同则并列
```

排名显示规则：

```text
并列后不跳号，例如 1、1、2、3。
```

## 6. 后台字段命名

后台所有展示文案中的：

```text
组别
```

统一改为：

```text
年龄组
```

实现建议：

- MVP 阶段可继续保留数据库字段 `group_name`。
- API 推荐新增或展示 `age_group` 语义字段。
- 旧参数 `group_name` 可作为兼容参数保留。

## 7. 体验优化点

针对约 20 个队、每周都有比赛的使用场景：

1. 球队下拉必须支持搜索。
2. 月份和日期联动，避免日期列表过长。
3. 比赛结果按日期分组展示。
4. 已选条件用筛选标签展示，支持单项清除。
5. 空状态提供“清空筛选”入口。
6. 移动端将筛选项折叠为筛选面板。
7. URL 同步筛选参数，便于分享查询链接。
