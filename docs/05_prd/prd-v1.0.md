# PRD v1.0

## 1. 产品名称

北京市活力网球交流系列赛赛果查看平台

## 2. 产品定位

本产品是一个面向单场网球交流赛的独立赛果积分查看平台。平台提供后台录入能力和公共端查询能力，用于支撑赛事期间和赛后的结果展示、球队排名展示、个人排名展示。

## 3. 目标用户

公共端用户：

- 参赛人员
- 球队成员
- 赛事组织者
- 关注赛事结果的人员

后台用户：

- 赛事管理员
- 成绩录入人员

## 4. 产品目标

MVP 需要达成：

- 后台可以录入球员、球队和成绩。
- 后台支持手工成绩录入和 Excel 上传。
- 公共端可以查看比赛结果。
- 公共端可以查看球队排名与积分。
- 公共端可以查看个人排名与积分。
- 公共端同时适配 Web 和手机端。

## 5. 功能范围

### 5.1 公共端功能

#### 5.1.1 比赛结果查询

用户可以查看本次交流赛的比赛结果。

展示内容建议包括：

- 项目或组别
- 结果类型或名次
- 球员或球队
- 积分
- 备注

本版不做详情页跳转。

#### 5.1.2 球队排名与积分

用户可以查看球队排名。

展示内容建议包括：

- 排名
- 球队名称
- 总积分
- 相关成绩数

排名规则：

- 按球队积分降序。
- 同分并列。
- 下一名跳号。
- 同名球队积分累加。

#### 5.1.3 个人排名与积分

用户可以查看个人排名。

展示内容建议包括：

- 排名
- 球员姓名
- 所属球队，若有
- 总积分
- 相关成绩数

排名规则：

- 按个人积分降序。
- 同分并列。
- 下一名跳号。
- 不按其他字段二次排序。

#### 5.1.4 搜索

公共端支持搜索：

- 球员
- 球队
- 项目或结果关键词

搜索只做关键词过滤，不做复杂筛选。

### 5.2 后台功能

#### 5.2.1 球员录入

后台用户可以：

- 新建球员
- 编辑球员
- 查看球员列表
- 搜索球员

#### 5.2.2 球队录入

后台用户可以：

- 新建球队
- 编辑球队
- 查看球队列表
- 搜索球队
- 维护球队成员

#### 5.2.3 成绩录入

后台用户可以通过两种方式录入成绩：

- 手工录入
- Excel 上传

成绩录入后，系统需要能支撑：

- 比赛结果查询
- 个人排名
- 球队排名

#### 5.2.4 成绩修正

后台用户可以查看已录入成绩并进行基础修正。

本版不做操作日志。

#### 5.2.5 排名预览

后台用户可以查看当前录入数据形成的个人排名和球队排名，便于确认录入效果。

## 6. 数据与计算规则

### 6.1 个人积分

- 个人积分只进入个人排名。
- 球队积分暂不分摊到个人。
- 个人成绩可以直接记录积分。

### 6.2 球队积分

- 球队积分只进入球队排名。
- 同名球队跨结果累加。
- 临时队伍也参与球队榜。

### 6.3 排名

- 按积分从高到低排序。
- 同分并列。
- 下一名跳号。
- 不做二次排序。

## 7. 响应式要求

公共端必须支持：

- 桌面浏览器
- 手机浏览器

桌面端：

- 排名可使用表格。
- 结果可使用表格或分组列表。

手机端：

- 排名使用卡片或紧凑列表。
- 不允许横向溢出。
- 搜索和 tab 入口需要清晰。

后台端优先支持桌面浏览器，手机端不作为后台核心验收项。

## 8. 非本版范围

本版不做：

- 多赛季
- 多赛事切换
- 球员主页
- 球队主页
- 数据导出
- 数据变更记录
- 操作日志
- 团体积分分摊到个人
- 与原年度积分系统直接集成
- 复杂权限系统

## 9. 验收标准

MVP 验收需要满足：

- 能新建球员。
- 能新建球队。
- 能维护球队成员。
- 能手工录入成绩。
- 能通过 Excel 上传成绩。
- 公共端能查看比赛结果。
- 公共端能查看球队排名。
- 公共端能查看个人排名。
- 公共端在手机端无明显布局问题。
- 前端展示内容来自后端真实接口。

## 10. 待确认项

- 是否启用后台登录。
- 成绩是否记录比分。
- Excel 上传模板字段。
- MVP 数据库是否使用 SQLite。
- 是否需要将“个人赛”和“积分赛”进一步定义为项目类型枚举。
