# 前端实施计划

## 1. 实施目标

前端需要实现：

- 公共端 Web 和手机端查询体验。
- 后台管理端基础录入体验。
- 所有展示数据来自后端 API。

## 2. 应用结构建议

建议采用一个前端项目，内部区分公共端和后台路由：

```text
frontend/
├─ src/
│  ├─ api/
│  ├─ components/
│  ├─ layouts/
│  ├─ views/
│  │  ├─ public/
│  │  └─ admin/
│  ├─ router/
│  ├─ styles/
│  └─ main.ts
```

如果后续希望公共端和后台分开部署，也可拆成两个 Vite 应用。

## 3. 公共端实现批次

### 批次 1：公共端框架

- 页面基础布局
- 赛事标题区
- 搜索框
- tab 导航
- 加载态、空状态、错误态

### 批次 2：比赛结果

- 比赛结果接口联调
- Web 表格/列表
- 手机端卡片
- 搜索过滤

### 批次 3：球队排名

- 球队排名接口联调
- 排名展示
- 同分排名展示
- 搜索过滤

### 批次 4：个人排名

- 个人排名接口联调
- 排名展示
- 同分排名展示
- 搜索过滤

## 4. 后台实现批次

### 批次 1：后台布局

- 后台布局
- 侧边导航
- 顶部栏
- 权限守卫，若启用登录

### 批次 2：球员管理

- 球员列表
- 新建球员
- 编辑球员
- 搜索球员

### 批次 3：球队管理

- 球队列表
- 新建球队
- 编辑球队
- 成员管理
- 搜索球队

### 批次 4：成绩录入

- 手工录入表单
- 个人结果录入
- 球队结果录入
- Excel 上传
- 上传预览
- 确认导入

### 批次 5：成绩管理和排名预览

- 成绩列表
- 编辑成绩
- 排名预览

## 5. 响应式要求

公共端必须适配：

- 375px 手机宽度
- 平板宽度
- 桌面宽度

后台端：

- 优先桌面端。
- 表格需要支持横向内容处理，但不能影响公共端。

## 6. 数据约束

- 不允许前端用假数据冒充后端能力。
- 接口未完成时，页面只能展示空状态或“接口未接入”的开发态说明。
- 排名计算必须由后端返回。
- 前端只负责展示和查询参数传递。

## 7. 验收检查

公共端：

- 比赛结果可查看。
- 球队排名可查看。
- 个人排名可查看。
- 搜索可用。
- 手机端不横向溢出。

后台：

- 能新增和编辑球员。
- 能新增和编辑球队。
- 能维护球队成员。
- 能手工录入成绩。
- 能上传 Excel 并确认导入。
- 能查看排名预览。
