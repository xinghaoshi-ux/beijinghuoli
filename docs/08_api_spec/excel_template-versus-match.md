# Excel 模板规范：队伍对抗赛局分制

## 1. 目标

本模板用于导入每一场具体的队伍对抗比赛。

不再导入：

```text
名次
固定积分
个人/团队结果范围
```

改为导入：

```text
双方队伍
双方球员
双方比分
场地
比赛日期
年龄组
```

胜方、球队累计局分、个人累计局分由系统自动计算。

## 2. 字段清单

| 列名 | 必填 | 示例 | 说明 |
|---|---|---|---|
| 比赛日期 | 是 | 2026-06-09 | 比赛实际发生日期，格式 `YYYY-MM-DD` |
| 场地 | 否 | 1号场地 | 可为空 |
| 年龄组 | 是 | 110岁组 | 可映射到比赛项目名称；旧模板中的“组别”可兼容解析 |
| 项目名称 | 否 | 110岁组双打 | 若为空，可用年龄组匹配项目 |
| 队伍A | 是 | 新赛道网球队 | 必须匹配已有球队 |
| 队伍A球员1 | 是 | 倪虹 | 必须匹配已有球员 |
| 队伍A球员2 | 条件必填 | 宋友春 | 双打必填 |
| 队伍A得分 | 是 | 5 | 非负整数 |
| 队伍B | 是 | 环保部网球队 | 必须匹配已有球队 |
| 队伍B球员1 | 是 | 杨朝飞 | 必须匹配已有球员 |
| 队伍B球员2 | 条件必填 | 崔铁成 | 双打必填 |
| 队伍B得分 | 是 | 3 | 非负整数 |
| 备注 | 否 | 加赛 | 可为空 |

## 3. 解析规则

### 3.1 比赛项目匹配

优先级：

```text
1. 如果填写项目名称，用项目名称匹配 CompetitionItem.name
2. 如果项目名称为空，用年龄组匹配 CompetitionItem.name
```

如果项目不存在：

```text
标记为错误行，不自动创建
```

### 3.2 球队匹配

```text
队伍A、队伍B 必须匹配已有 Team.name
```

校验：

```text
队伍A != 队伍B
```

### 3.3 球员匹配

```text
球员姓名必须匹配已有 Player.name
```

MVP 建议：

```text
球员必须属于对应球队，或当前 TeamMember 中存在有效绑定
```

如果球员不存在或不属于该队：

```text
标记为错误行
```

### 3.4 球员数量校验

使用 `CompetitionItem.player_count` 校验每侧球员数量。

示例：

```text
player_count = 2
队伍A球员1、队伍A球员2 必填
队伍B球员1、队伍B球员2 必填
```

### 3.5 比分校验

```text
队伍A得分 >= 0
队伍B得分 >= 0
队伍A得分 != 队伍B得分
```

胜方由系统自动计算：

```text
队伍A得分 > 队伍B得分，则队伍A胜
队伍B得分 > 队伍A得分，则队伍B胜
```

## 5. 行状态

| 值 | 含义 |
|---|---|
| valid | 可导入 |
| warning | 可导入但需关注 |
| error | 不可导入 |

错误示例：

- 项目不存在。
- 球队不存在。
- 球员不存在。
- 球员不属于对应球队。
- 每侧球员数量不符合项目要求。
- 队伍A和队伍B相同。
- 比分为空。
- 比分不是非负整数。
- 双方比分相同。

## 6. 导入确认

确认导入接口沿用：

```text
POST /admin/uploads/{upload_id}/confirm
```

请求：

```json
{
  "confirmed_rows": [2, 3, 4]
}
```

导入后生成：

```text
VersusMatch
VersusMatchPlayer
```

不再生成手动积分记录。

## 7. 暂不支持

MVP 暂不支持：

- Excel 中自动创建球队。
- Excel 中自动创建球员。
- Excel 中自动创建比赛项目。
- 平局比分。
- 一行内录入多场比赛。
