# Excel 模板草案

## 1. 目标

本文定义 MVP 阶段 Excel 成绩上传模板草案。

当前模板用于支持单场交流赛成绩导入。最终字段可在进入后端开发前确认。

## 2. 模板策略

MVP 建议使用一套统一模板，同时支持个人结果和球队结果。

优点：

- 后台上传入口简单。
- 管理员只需理解一套字段。
- 后端可根据 `结果范围` 判断个人或球队。

## 3. 字段清单

| 列名 | 必填 | 示例 | 说明 |
|---|---|---|---|
| 项目名称 | 是 | 男子双打 | 对应 CompetitionItem.name |
| 结果范围 | 是 | individual | `individual` 或 `team` |
| 名次标签 | 否 | 冠军 | 冠军、亚军、第 1 名等 |
| 名次排序 | 否 | 1 | 用于排序 |
| 球员1 | 条件必填 | 张三 | 个人结果至少填写球员1 |
| 球员2 | 否 | 李四 | 双打或多人结果使用 |
| 球员3 | 否 | 王五 | 可选 |
| 球员4 | 否 | 赵六 | 可选 |
| 球队名称 | 条件必填 | 住建部队 | 球队结果必填 |
| 比分 | 否 | 6:4 | 可为空 |
| 积分 | 是 | 100 | 非负整数 |
| 备注 | 否 | 决赛 | 可为空 |

## 4. 解析规则

### 4.1 个人结果

当 `结果范围` 为 `individual`：

- 必须填写 `项目名称`。
- 必须填写至少一个球员。
- `球队名称` 可为空。
- 系统根据球员姓名匹配 Player。
- 导入后写入 MatchResult、MatchResultPlayer、PlayerPoints。

### 4.2 球队结果

当 `结果范围` 为 `team`：

- 必须填写 `项目名称`。
- 必须填写 `球队名称`。
- 球员字段可为空。
- 系统根据球队名称匹配 Team。
- 导入后写入 MatchResult、TeamPoints。
- 不写入 PlayerPoints。

## 5. 匹配规则

MVP 建议：

- Excel 中不存在的球员不自动创建，标记为错误行。
- Excel 中不存在的球队不自动创建，标记为错误行。
- Excel 中不存在的项目不自动创建，标记为错误行。

原因：

- 避免姓名错别字造成脏数据。
- 后台已提供球员、球队、项目维护入口。

如后续希望提升导入效率，可增加“允许自动创建”的导入选项。

## 6. 行状态

解析预览行状态：

| 值 | 含义 |
|---|---|
| normal | 正常 |
| warning | 警告，可导入 |
| error | 错误，不可导入 |

错误示例：

- 项目不存在。
- 球员不存在。
- 球队不存在。
- 积分为空。
- 积分不是数字。
- 结果范围无效。

## 7. 待确认项

- 是否需要拆分个人结果模板和球队结果模板。
- 是否允许自动创建不存在的球员、球队、项目。
- 球员列最大支持几名球员。
