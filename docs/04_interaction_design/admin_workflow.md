# 后台管理流程

## 1. 目标

后台用于赛事管理员完成基础数据维护与成绩录入。

核心任务：

- 录入球员
- 录入球队
- 维护球队成员
- 录入成绩
- 上传 Excel 成绩
- 查看排名预览

后台优先支持桌面浏览器。

## 2. 后台入口

是否启用后台登录仍为待确认项。

建议方案：

- MVP 如需上线给多人使用，应启用后台登录。
- 若仅本地单人录入，可暂缓复杂权限，只保留一个管理员账号。

## 3. 后台导航

后台建议包含以下一级菜单：

- 概览
- 球员管理
- 球队管理
- 成绩录入
- 成绩管理
- 排名预览

## 4. 球员管理流程

### 4.1 查看球员

1. 管理员进入球员管理。
2. 系统展示球员列表。
3. 管理员可按姓名搜索。

列表字段：

- 姓名
- 默认球队，若有
- 状态
- 备注
- 操作

### 4.2 新建球员

1. 管理员点击新建。
2. 填写姓名、默认球队、联系方式、备注。
3. 保存。
4. 系统返回列表并刷新数据。

### 4.3 编辑球员

1. 管理员点击编辑。
2. 修改信息。
3. 保存。
4. 系统刷新列表。

## 5. 球队管理流程

### 5.1 查看球队

1. 管理员进入球队管理。
2. 系统展示球队列表。
3. 管理员可按球队名称搜索。

列表字段：

- 球队名称
- 成员数
- 状态
- 备注
- 操作

### 5.2 新建球队

1. 管理员点击新建。
2. 填写球队名称、备注。
3. 保存。
4. 系统刷新列表。

### 5.3 维护球队成员

1. 管理员在球队列表点击成员管理。
2. 系统展示该球队已选成员。
3. 管理员添加或移除球员。
4. 保存。
5. 系统更新 TeamMember。

## 6. 成绩录入流程

成绩录入分为：

- 手工录入
- Excel 上传

### 6.1 手工录入成绩

流程：

1. 管理员进入成绩录入。
2. 选择比赛项目。
3. 选择结果范围：个人或球队。
4. 填写名次或成绩标签。
5. 选择球员或球队。
6. 填写积分。
7. 填写比分和备注，若有。
8. 保存。
9. 系统写入 MatchResult，并同步写入 PlayerPoints 或 TeamPoints。

个人结果：

- 选择 1 名或多名球员。
- 保存后写入 PlayerPoints。

球队结果：

- 选择 1 个球队。
- 保存后写入 TeamPoints。
- 不分摊给个人。

### 6.2 Excel 上传成绩

流程：

1. 管理员进入成绩录入。
2. 选择 Excel 上传。
3. 上传文件。
4. 系统解析并展示预览。
5. 管理员检查正常行、错误行。
6. 管理员确认导入。
7. 系统写入结果和积分。

预览页需要展示：

- 行号
- 项目名称
- 结果范围
- 球员或球队
- 积分
- 比分
- 状态
- 错误说明

## 7. 成绩管理流程

### 7.1 查看成绩列表

1. 管理员进入成绩管理。
2. 系统展示已录入成绩。
3. 管理员可搜索关键词。

### 7.2 编辑成绩

1. 管理员点击编辑。
2. 修改结果信息或积分。
3. 保存。
4. 系统同步更新对应 PlayerPoints 或 TeamPoints。

### 7.3 删除成绩

是否允许删除成绩待确认。

如果允许：

- 删除 MatchResult。
- 删除关联 MatchResultPlayer。
- 删除对应 PlayerPoints 或 TeamPoints。

## 8. 排名预览流程

1. 管理员进入排名预览。
2. 系统展示个人排名和球队排名。
3. 管理员核对录入效果。
4. 如发现问题，返回成绩管理修正。

## 9. 状态设计

后台需要具备：

- 加载态
- 保存中状态
- 上传中状态
- 解析中状态
- 空状态
- 错误提示
- 表单校验提示

## 10. 明确不做

本版后台不做：

- 多角色权限
- 操作日志
- 数据导出
- 多赛事管理
- 多赛季管理
- 复杂工作流审批
