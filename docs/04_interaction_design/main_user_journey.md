# 公共端用户旅程

## 1. 目标

公共端用于让参赛者和关注赛事的人快速查看：

- 比赛结果
- 球队排名与积分
- 个人排名与积分

公共端不需要登录，必须支持 Web 端和手机端。

## 2. 页面入口

公共端首屏即为赛事查询页面，不做营销式首页。

首屏需要包含：

- 赛事名称
- 搜索框
- 三个功能 tab：
  - 比赛结果
  - 球队排名
  - 个人排名

## 3. 默认展示

默认进入页面时：

1. 加载赛事基础信息。
2. 默认打开“比赛结果”tab。
3. 展示最新或按项目排序的比赛结果。
4. 同时允许用户切换到球队排名或个人排名。

## 4. 用户主流程

### 4.1 查看比赛结果

流程：

1. 用户打开公共端。
2. 默认进入比赛结果 tab。
3. 系统展示结果列表。
4. 用户可以输入关键词搜索。
5. 系统按关键词过滤结果。

展示字段建议：

- 项目或组别
- 名次或成绩标签
- 球员或球队
- 比分，若有
- 积分
- 备注，若有

本版不做结果详情页。

### 4.2 查看球队排名

流程：

1. 用户点击“球队排名”tab。
2. 系统展示球队排名。
3. 用户可以搜索球队名称。
4. 系统过滤球队排名结果。

展示字段建议：

- 排名
- 球队名称
- 总积分
- 成绩数

本版不做球队主页。

### 4.3 查看个人排名

流程：

1. 用户点击“个人排名”tab。
2. 系统展示个人排名。
3. 用户可以搜索球员姓名。
4. 系统过滤个人排名结果。

展示字段建议：

- 排名
- 球员姓名
- 所属球队，若有
- 总积分
- 成绩数

本版不做球员详情弹窗或个人主页。

## 5. 搜索行为

搜索框为公共端统一搜索入口。

不同行为：

- 比赛结果 tab：搜索球员、球队、项目、成绩标签、备注。
- 球队排名 tab：搜索球队名称。
- 个人排名 tab：搜索球员姓名。

搜索规则：

- 关键词为空时展示全部。
- 输入后触发过滤或查询。
- 不做高级筛选。
- 不做院系、赛事级别等筛选。

## 6. 状态设计

### 6.1 加载态

进入页面或切换 tab 时，如数据未返回，展示加载状态。

### 6.2 空状态

无数据时，根据 tab 展示：

- 比赛结果：暂无比赛结果
- 球队排名：暂无球队排名
- 个人排名：暂无个人排名

### 6.3 错误态

接口异常时展示简短错误信息，并提供刷新入口。

### 6.4 搜索无结果

展示“未找到匹配结果”，并允许清空搜索。

## 7. 手机端要求

手机端：

- 搜索框和 tab 必须在首屏可见。
- 排名使用卡片或紧凑列表。
- 比赛结果使用分组列表或卡片。
- 不允许横向滚动。
- 单行文本过长时允许换行或省略。

## 8. 明确不做

本版公共端不做：

- 球员详情弹窗
- 球员主页
- 球队主页
- 多赛季切换
- 多赛事切换
- 数据导出
- 高级筛选
