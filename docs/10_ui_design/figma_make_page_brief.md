# Figma Make 修改说明：北京市活力网球交流系列赛赛果查看平台

只新增和设计以下页面，不要做多赛季、多赛事、球员主页、球队主页、数据导出、操作日志页面。

整体视觉：
- 公共端按参考图风格设计。
- 背景使用浅紫灰。
- 主色使用紫色。
- 强调色使用荧光黄绿色。
- 主标题使用深紫黑色。
- 卡片使用白色、轻阴影、18px 左右大圆角。
- 搜索框、tab、表格使用浅紫色描边。
- Header、空状态和内容卡片角落可以放少量网球、球拍、曲线线稿装饰。
- 积分数字使用绿色强调。
- 公共端必须同时适配 Web 和手机端。
- 后台端使用工作台式布局，左侧导航 + 顶部栏 + 主内容区。

---

## 1. 新增公共端 `PublicHome`

页面用途：给参赛者查看比赛结果、球队排名、个人排名。

路径：`/`

### 1.1 顶部赛事 Header

位置：页面最顶部。

内容：
- 主标题：`北京市活力网球交流系列赛`
- 副标题：`赛果积分查看平台`
- 右侧或标题下方放一个小标签：`公开查询`

样式：
- 桌面端标题居左，内容宽度居中。
- 手机端标题换行展示，不要横向溢出。
- 按参考图做白色大圆角 Header 卡片。
- Header 右侧放淡黄色网球和浅紫色曲线装饰。
- 右上角放荧光黄绿色胶囊按钮，文字：`公开查询`。

### 1.2 搜索区

位置：Header 下方。

内容：
- 搜索输入框，占据主要宽度。
- placeholder：`搜索球员、球队、项目或结果`
- 右侧放搜索图标按钮。

点击关系：
- 输入关键词后，当前 tab 内过滤结果。
- 清空关键词后恢复全部数据。

### 1.3 三个 Tab

位置：搜索区下方。

Tab：
- `比赛结果`
- `球队排名`
- `个人排名`

默认：
- 默认选中 `比赛结果`。

样式：
- 桌面端使用参考图的横向三段式 segmented tab。
- 手机端三个 tab 等宽排列。
- 当前 tab 使用紫色实底，文字白色，可在底部加小三角指示。

点击关系：
- 点击 `比赛结果` → 显示比赛结果列表。
- 点击 `球队排名` → 显示球队排名列表。
- 点击 `个人排名` → 显示个人排名列表。

### 1.4 比赛结果列表

位置：Tab 下方，`比赛结果` tab 内容区。

桌面端内容：
- 使用参考图风格表格。
- 列：`项目`、`名次/成绩`、`球员/球队`、`比分`、`积分`、`备注`
- 表头使用浅紫底色。
- 表格外层放在白色圆角内容卡片内。
- 卡片标题左侧放荧光黄绿色圆形奖杯图标。

手机端内容：
- 不照搬桌面表格，改为结果卡片。
- 每张卡片从上到下：
  1. 项目名称，例如 `男子双打`
  2. 名次标签，例如 `冠军`
  3. 球员组合或球队名称
  4. 比分，若没有则不显示
  5. 积分，例如 `100 分`
  6. 备注，若没有则不显示

点击关系：
- 本版不进入详情页。
- 点击卡片不跳转。

空状态：
- 文案：`暂无比赛结果`

### 1.5 球队排名列表

位置：Tab 下方，`球队排名` tab 内容区。

桌面端内容：
- 使用排名表格。
- 列：`排名`、`球队`、`总积分`、`成绩数`

手机端内容：
- 使用排名卡片。
- 每张卡片从左到右：
  1. 排名，例如 `#1`
  2. 球队名称
  3. 成绩数，例如 `3 条成绩`
  4. 总积分，例如 `300 分`

样式：
- 前三名排名数字可以使用更明显的主色。
- 同分并列时直接显示相同排名。
- 保持参考图的紫色 tab、白色卡片、绿色积分数字。

点击关系：
- 本版不进入球队主页。
- 点击卡片不跳转。

空状态：
- 文案：`暂无球队排名`

### 1.6 个人排名列表

位置：Tab 下方，`个人排名` tab 内容区。

桌面端内容：
- 使用排名表格。
- 列：`排名`、`球员`、`所属球队`、`总积分`、`成绩数`

手机端内容：
- 使用排名卡片。
- 每张卡片从左到右：
  1. 排名，例如 `#1`
  2. 球员姓名
  3. 所属球队，若没有则显示 `未归属球队`
  4. 总积分，例如 `180 分`

点击关系：
- 本版不做球员详情弹窗。
- 本版不进入球员主页。
- 点击卡片不跳转。

空状态：
- 文案：`暂无个人排名`

## 7. 新增后台 `AdminResultEntry`

页面用途：录入成绩，支持手工录入和 Excel 上传。

路径：`/admin/results/new`

### 7.1 录入方式切换

位置：页面顶部。

Tab：
- `手工录入`
- `Excel 上传`

默认：
- 选中 `手工录入`

### 7.2 手工录入表单

位置：`手工录入` tab 内容区。

表单从上到下：
1. 比赛项目选择器
2. 结果范围选择：`个人 / 球队`
3. 名次标签输入框，例如 `冠军`
4. 名次排序数字输入框
5. 个人结果时显示：球员多选框
6. 球队结果时显示：球队选择器
7. 比分输入框
8. 积分数字输入框
9. 备注输入框
10. 主按钮：`保存成绩`

点击关系：
- 选择 `个人` → 显示球员多选，不显示球队选择
- 选择 `球队` → 显示球队选择，不显示球员多选
- 点击 `保存成绩` → 保存后跳转或提示，并清空表单

### 7.3 Excel 上传区

位置：`Excel 上传` tab 内容区。

内容：
- 上传区域
- 文案：`上传 .xlsx 成绩文件`
- 次级说明：`请使用统一成绩模板`
- 主按钮：`上传并解析`

点击关系：
- 上传文件后点击 `上传并解析` → 显示解析预览表

### 7.4 Excel 解析预览表

位置：上传区下方。

列：
- `行号`
- `项目名称`
- `结果范围`
- `球员/球队`
- `比分`
- `积分`
- `状态`
- `错误说明`

底部按钮：
- `取消上传`
- `确认导入`

点击关系：
- 点击 `确认导入` → 导入正常行，成功后进入 `AdminResults`
- 点击 `取消上传` → 清空当前上传状态

## 10. 复用说明

公共端：
- 公共端所有列表卡片复用参考图的白色大圆角卡片样式。
- `比赛结果`、`球队排名`、`个人排名` 共用参考图的搜索框和三段式 tab 样式。
- 手机端排名卡片复用同一布局，只替换名称字段和积分字段。
- 空状态复用参考图的球拍、网球、浅紫曲线装饰。

后台端：
- `AdminPlayers`、`AdminTeams`、`AdminResults` 复用同一套“顶部操作栏 + 表格 + 弹窗”的页面结构。
- `AdminResultEntry` 和 `AdminResults` 的编辑弹窗复用同一套成绩表单。
- `AdminRankings` 的两个排名表格复用同一套排名表格样式。

不要新增：
- 球员主页
- 球队主页
- 数据导出入口
- 操作日志页面
- 多赛季或多赛事切换入口
