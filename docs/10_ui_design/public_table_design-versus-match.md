# 公共端 Table 设计稿：队伍对抗赛局分制

## 1. 设计结论

当前公共端三张表需要从“名次积分制”改为“队伍对抗 + 局分累计制”。

现有表格不再适合：

```text
项目 | 名次/成绩 | 球员/球队 | 比分 | 积分 | 备注
```

新的公共端应保留三个 Tab：

```text
比赛结果
球队排名
个人排名
```

但每个 Tab 内的 table 字段需要重排。

---

## 2. 公共端筛选区

### 2.1 位置

筛选区放在顶部标题卡片内，搜索框下方或同一行右侧，根据屏幕宽度自适应换行。

比赛结果 Tab 桌面端推荐顺序：

```text
搜索框 | 月份下拉 | 比赛日期下拉 | 球队下拉 | 年龄组下拉 | 重置
```

球队排名 / 个人排名 Tab 桌面端推荐顺序：

```text
搜索框 | 球队下拉 | 重置
```

手机端推荐：

```text
搜索框
筛选按钮
已选条件标签
```

点击“筛选”后展开：

```text
月份
比赛日期
球队
年龄组
重置
确认
```

### 2.2 筛选项

| 筛选项 | 控件 | 数据来源 | 说明 |
|---|---|---|---|
| 搜索 | 输入框 | 用户输入 | 搜索球员、球队、项目或结果 |
| 月份 | 下拉菜单 | `/public/filter-options.match_months` | 仅比赛结果 Tab 展示，格式 `YYYY-MM` |
| 比赛日期 | 下拉菜单 | `/public/filter-options.match_dates` | 仅比赛结果 Tab 展示，格式 `YYYY-MM-DD` |
| 球队 | 下拉菜单 | `/public/filter-options.teams` | 比赛结果、球队排名、个人排名均可使用 |
| 年龄组 | 下拉菜单 | `/public/filter-options.age_groups` | 仅比赛结果 Tab 展示 |

### 2.3 交互规则

- 默认显示最近一个有比赛结果的比赛日期。
- 选择月份后，比赛日期下拉只显示该月份内有比赛的日期。
- 选择球队后，比赛结果只显示该球队参与的比赛。
- 球队排名和个人排名只展示最新全局总榜，不展示月份、比赛日期、年龄组筛选。
- 球队排名和个人排名选择球队后只缩小展示范围，排名数字仍保留全局名次。
- 已选择条件用标签展示，支持单项清除。
- 点击“重置”恢复到默认最近比赛日。
- URL 可同步筛选参数，方便分享某一天或某个队的查询结果。

### 2.4 20 个队体验优化

- 球队下拉支持输入搜索，不展示长列表铺满页面。
- 球队选项同时展示全称和简称，如 `新赛道网球队 / 新赛道`。
- 最近选择的球队可置顶，但必须来自后端真实数据。
- 比赛结果列表按日期分组，避免每周比赛累积后信息密度过高。
- 空状态文案要提示当前筛选条件下无结果，并提供“清空筛选”。

## 3. 比赛结果 Table

### 3.1 页面用途

让用户快速看懂每一场具体对抗：

```text
谁和谁打
双方球员是谁
比分是多少
谁赢
```

### 3.2 桌面端表头

```text
日期 | 场地 | 年龄组 | 对阵 | 比分 | 胜方
```

### 3.3 字段说明

| 列 | 内容 | 示例 |
|---|---|---|
| 日期 | 比赛日期 | `2026.06.09` |
| 场地 | 比赛场地 | `1号场地` |
| 年龄组 | 比赛年龄组 | `110岁组` |
| 对阵 | 队伍和球员组合 | `新赛道：倪虹 / 宋友春 vs 环保部：杨朝飞 / 崔铁成` |
| 比分 | 队伍 A 得分 : 队伍 B 得分 | `5 : 3` |
| 胜方 | 系统根据比分自动计算 | `新赛道` |

### 3.4 推荐行展示

```text
2026.06.09 | 1号场地 | 110岁组 | 新赛道：倪虹 / 宋友春 vs 环保部：杨朝飞 / 崔铁成 | 5 : 3 | 新赛道
```

### 3.5 手机端卡片

手机端不使用横向 table，改成卡片。

每张卡片从上到下：

```text
2026.06.09 · 第 1 场 · 1号场地 · 110岁组
新赛道
倪虹 / 宋友春
5 : 3
环保部
杨朝飞 / 崔铁成
胜方：新赛道
```

### 3.6 空状态

文案：

```text
暂无比赛结果
```

空状态只显示图 6 的球拍网球贴图，不再放左右两根虚线。

## 4. 球队排名 Table

### 4.1 页面用途

展示团体对抗排名。

这里的排名不是按单场胜负直接排，而是按队伍之间总比分形成的团体战绩排序。

球队排名展示最新全局总榜。可以通过球队下拉缩小展示范围，但排名数字仍保留全局总榜名次，不因为筛选重新计算。

### 4.2 桌面端表头

```text
排名 | 球队 | 团体战绩 | 胜盘 | 胜局 | 净胜局
```

### 4.3 字段说明

| 列 | 内容 | 示例 |
|---|---|---|
| 排名 | 系统自动计算 | `1` |
| 球队 | 球队名称 | `新赛道网球队` |
| 团体战绩 | 按队伍间总比分计算胜负 | `2胜0负` |
| 胜盘 | 队伍获胜盘数，每条比赛记录为 1 盘 | `5` |
| 胜局 | 队伍累计本方局分 | `31` |
| 净胜局 | 本方胜局减对方胜局 | `+14` |

### 4.4 推荐行展示

```text
1 | 新赛道网球队 | 2胜0负 | 5 | 31 | +14
2 | 环保部网球队 | 1胜1负 | 3 | 24 | 0
3 | 人社部网球队 | 0胜2负 | 1 | 17 | -14
```

### 4.5 手机端卡片

每张卡片从左到右/从上到下：

```text
#1 新赛道网球队
团体战绩：2胜0负
胜盘：5
胜局：31
净胜局：+14
```

### 4.6 排序规则

```text
1. 团体胜场数
2. 胜盘数
3. 胜局数
4. 净胜局
5. 仍相同则并列
```

并列后不跳号，例如：

```text
1
1
2
3
```

## 5. 个人排名 Table

### 5.1 页面用途

展示球员个人排名。个人排名按胜盘、胜局、净胜局、出场次数排序。

个人排名展示最新全局总榜。

```text
搜索框 | 球队下拉 | 重置
```

说明：

- 默认展示全局排名。
- 选择球队后只缩小展示范围，用于查看该队球员。
- 筛选后排名数字仍保留全局总榜名次，不重新计算队内排名。
- 个人排名不展示月份、比赛日期、年龄组筛选，也不提供“全局排名 / 队内排名”切换。

### 5.2 桌面端表头

```text
排名 | 球员 | 所属队伍 | 出场 | 胜盘 | 胜局 | 净胜局
```

### 5.3 字段说明

| 列 | 内容 | 示例 |
|---|---|---|
| 排名 | 系统自动计算 | `1` |
| 球员 | 球员姓名 | `倪虹` |
| 所属队伍 | 球员所在队伍 | `新赛道网球队` |
| 出场 | 该球员参赛场数 | `2` |
| 胜盘 | 该球员所在一方获胜盘数 | `2` |
| 胜局 | 个人出场比赛中本方累计局分 | `12` |
| 净胜局 | 本方胜局减对方胜局 | `+8` |

### 5.4 推荐行展示

```text
1 | 倪虹 | 新赛道网球队 | 2 | 2 | 12 | +8
1 | 宋友春 | 新赛道网球队 | 2 | 2 | 12 | +8
3 | 陈伟 | 新赛道网球队 | 2 | 2 | 11 | +6
3 | 韩凯 | 新赛道网球队 | 2 | 2 | 11 | +6
```

### 5.5 手机端卡片

```text
#1 倪虹
新赛道网球队
出场 2 · 胜盘 2
胜局 12 · 净胜局 +8
```

### 5.6 排序规则

```text
1. 个人胜盘数
2. 个人胜局数
3. 个人净胜局
4. 出场次数
5. 仍相同则并列
```

并列后不跳号，例如：

```text
1
1
2
3
```

## 9. 不在本轮 table 设计中处理

本设计稿只定义公共端 table 展示方式，不直接处理：

- 后台录入表单实现。
- Excel 模板字段。
- 数据库迁移细节。
- 旧数据迁移。
- 排名缓存表。

这些需要在数据模型和 API 契约确认后再进入实现。
