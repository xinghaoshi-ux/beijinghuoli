# MVP 范围说明

## 1. 背景

本项目用于建设“北京市活力网球交流系列赛”的独立赛果查看平台。

该平台服务于单场交流赛，不承担年度积分系统、多赛事系统或长期联赛系统的职责。当前 MVP 聚焦让赛事管理员能够录入基础数据和成绩，让参赛者能够在 Web 或手机端查看比赛结果、球队排名与个人排名。

## 2. 核心问题

当前需要解决的问题：

- 比赛结果需要有统一查询入口。
- 球队积分和个人积分需要清晰分开展示。
- 后台需要支持录入球队、球员和成绩。
- 公共端需要适配 Web 和手机端。
- 项目需要保持独立，避免影响原年度积分平台。

## 3. MVP 目标

MVP 需要完成以下闭环：

1. 管理员录入球员。
2. 管理员录入球队。
3. 管理员维护球队成员。
4. 管理员通过手工录入或 Excel 上传录入成绩。
5. 系统根据成绩形成个人排名和球队排名。
6. 公共端展示比赛结果、球队排名、个人排名。
7. 公共端支持 Web 和手机端访问。

## 4. MVP 范围

### 4.1 公共端

本版公共端包含：

- 比赛结果查询
- 球队排名与积分
- 个人排名与积分
- 搜索球员、球队、项目或结果关键词
- Web 端和手机端响应式展示

### 4.2 后台

本版后台包含：

- 后台登录，是否启用待 Phase 3/5 最终确认
- 球员录入、编辑、查询
- 球队录入、编辑、查询
- 球队成员维护
- 成绩手工录入
- Excel 成绩上传
- 成绩列表查看与基础修正
- 排名预览

### 4.3 排名规则

本版排名规则：

- 排名分为个人排名和球队排名。
- 个人积分和球队积分分开计算。
- 团体或球队积分暂不分摊到个人。
- 同分并列，下一名跳号。
- 不做参赛次数、冠军数、最近成绩等二次排序。

## 5. 非 MVP 范围

本版不做：

- 多赛季
- 多赛事切换
- 球员个人主页
- 球队主页
- 数据导出
- 数据变更记录
- 操作日志
- 团体积分分摊到个人
- 复杂筛选条件
- 与原年度积分平台直接集成
- 多角色权限系统
- 微信、小程序或原生 App

## 6. 用户角色

### 6.1 公共端用户

公共端用户可以：

- 查看比赛结果
- 查看球队排名
- 查看个人排名
- 搜索球员、球队或结果关键词

公共端用户不需要登录。

### 6.2 后台用户

后台用户可以：

- 维护球员
- 维护球队
- 维护球队成员
- 录入成绩
- 上传 Excel
- 查看录入后的排名结果

## 7. 主链路

本版主链路：

1. 后台录入球员。
2. 后台录入球队。
3. 后台维护球队成员。
4. 后台录入比赛结果。
5. 系统生成或保存积分。
6. 公共端展示比赛结果。
7. 公共端展示个人排名和球队排名。

## 8. 待确认项

- “个人赛”和“积分赛”的业务定义是否需要进一步区分为项目类型。
- 成绩是否需要记录比分，例如 6:4、7:5。
- Excel 模板字段是否固定为一套，还是个人赛和球队赛分别一套。
- 后台是否必须登录。
- MVP 数据库是否确定使用 SQLite。
