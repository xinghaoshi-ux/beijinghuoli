# Phase 6 后端实现说明

## 1. 实现范围

本阶段完成后端 MVP 主链路实现。

已实现：

- FastAPI 后端项目骨架
- SQLite 数据库连接
- 启动时自动建表
- 初始管理员创建
- 后台登录与 Token 验证
- 球员管理 API
- 球队管理 API
- 球队成员维护 API
- 比赛项目 API
- 手工成绩录入 API
- 成绩编辑与删除 API
- Excel 上传、解析、预览、确认导入 API
- 公共端赛事信息 API
- 公共端比赛结果 API
- 公共端个人排名 API
- 公共端球队排名 API
- 后台排名预览 API
- 后台概览 API

## 2. 技术实现

后端技术：

- FastAPI
- SQLAlchemy async
- SQLite
- Pydantic
- python-jose
- passlib
- openpyxl
- pytest

数据库策略：

- MVP 使用 SQLite。
- 启动时自动建表。
- 暂未引入 Alembic。

认证策略：

- 默认启用后台登录。
- 默认初始账号来自环境变量。
- 默认用户名：`admin`
- 默认密码：`admin123`

## 3. 核心规则实现

已按文档实现：

- 个人积分和球队积分分表存储。
- 球队积分不分摊到个人。
- 成绩录入后同步生成积分记录。
- 成绩编辑后重新生成对应积分记录。
- 个人排名按 PlayerPoints 聚合。
- 球队排名按 TeamPoints 聚合。
- 同分并列，下一名跳号。
- 公共端不需要登录。
- 后台接口默认需要登录。

## 4. Excel 上传实现

当前 Excel 上传支持统一模板：

- 项目名称
- 结果范围
- 名次标签
- 名次排序
- 球员1
- 球员2
- 球员3
- 球员4
- 球队名称
- 比分
- 积分
- 备注

解析规则：

- 不自动创建不存在的球员。
- 不自动创建不存在的球队。
- 不自动创建不存在的比赛项目。
- 错误行不可导入。

## 5. 后续注意

- 若后续需要长期部署，建议增加 Alembic 迁移。
- 若后台多人使用，建议进一步完善密码修改和账号管理。
- 若 Excel 模板变化，需要同步更新 API 文档和解析逻辑。
