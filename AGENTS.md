# AGENTS.md

## 项目目标

本项目用于建设“住建部、人社部、环保部网球交流赛”的独立赛果积分查看平台。

当前阶段只进行项目初始化、边界确认、文档规范和后续开发规则搭建，不进入业务代码实现。

## 项目边界

本项目是独立项目，不直接升级或污染原有 `Tennis points platform` 年度积分系统。

本版只服务单场交流赛，支持：

- 公共端比赛结果查询
- 公共端球队排名与积分
- 公共端个人排名与积分
- 后台球队录入
- 后台球员录入
- 后台成绩录入

本版不做：

- 多赛季
- 多赛事切换
- 球员主页
- 球队主页
- 数据导出
- 数据变更记录
- 团体积分分摊到个人
- 与原年度系统直接集成

## 目录职责

### `frontend/`

- 承载公共端和后台管理端前端实现。
- 负责页面、交互、状态管理、API 调用和构建。
- 不得用假数据冒充后端真实能力。

### `backend/`

- 承载后端 API、业务逻辑、数据访问、鉴权和测试。
- 不得在未更新 API 文档的情况下擅自改变接口契约。

### `docs/`

- 承载项目规则、需求、交互、架构、数据模型、API、联调和部署文档。
- 是实现的上游依据。

## 执行原则

- 复杂任务先计划再执行。
- 所有业务实现必须基于已确认的 `docs/` 文档。
- API 契约必须先于前后端联调。
- MVP 先完成主链路，再做增强能力。
- 未确认内容不得写成既定事实。

## 阶段要求

Phase 1 只允许创建项目初始化与规则文档，不写业务代码。

后续阶段建议：

1. Phase 1：项目初始化与规则
2. Phase 2：需求收口与 MVP 范围
3. Phase 3：业务规则与数据模型
4. Phase 4：交互流程与页面规划
5. Phase 5：API 契约设计
6. Phase 6：后端 MVP 开发
7. Phase 7：前端 MVP 开发
8. Phase 8：联调、测试与部署
